Section 01
Core Framework

The foundation every other library sits on top of.

React 18
UI Library

Definition: React is a JavaScript library for building user interfaces using a component-based architecture. It uses a virtual DOM to efficiently update only the parts of the real DOM that changed, making UIs fast and predictable.

Why I Used It

React is the industry standard for SPAs. Its component model allowed me to build every section (Hero, About, Skills, etc.) as a self-contained, reusable unit. React 18's concurrent features (like useTransition) also make animations smoother by keeping the UI responsive during heavy updates. It's what I use daily in production.

Component-based Virtual DOM JSX Hooks SPA
Vite 5
Build Tool

Definition: Vite is a next-generation frontend build tool. It uses native ES Modules in the browser during development (no bundling step needed), making the dev server start almost instantly. For production it uses Rollup for optimized bundling.

Why I Used It

Compared to Create React App (which uses Webpack), Vite's dev server starts in under 300ms regardless of project size. HMR (Hot Module Replacement) is nearly instant — when I change a component, the browser updates in milliseconds without losing state. Production builds are also 30–40% faster. For a portfolio, fast feedback loops during development are critical.

ESM-based dev server HMR Rollup production build Fast cold start
Tailwind CSS 3
Styling

Definition: Tailwind is a utility-first CSS framework. Instead of writing custom CSS classes, you compose designs directly in your HTML/JSX using small utility classes like flex, text-lg, rounded-xl. It purges unused CSS in production, resulting in tiny stylesheets.

Why I Used It

Tailwind eliminates the context-switching between JSX and CSS files. I extended it with custom design tokens for the portfolio's exact color palette (bg-bg-primary, text-accent, etc.), so every component uses consistent values from a single config. The production CSS bundle came out at just ~27KB — extremely lean.

Utility-first Design tokens PurgeCSS Responsive Dark theme
Key Interview Point: "I chose React + Vite over Next.js intentionally. A portfolio is a purely static single-page app — there's no need for SSR, API routes, or server components. Vite gives me the same DX as Next.js but with a simpler, smaller setup and faster builds."

GSAP — GreenSock Animation Platform
Animation Engine

Definition: GSAP is a JavaScript animation library known for its performance, precision, and timeline control. It animates anything — DOM elements, CSS properties, SVG, canvas — at 60fps even on low-end devices. It does not use React's render cycle, which is exactly what makes it fast.

Why I Used It

Framer Motion is convenient but operates inside React's render cycle. For heavy, chained animations — like the Hero page load sequence (badge → name char-by-char → tagline → CTA buttons) or the Experience timeline line that draws as you scroll — GSAP's timeline API gives precise frame-level control that React state simply cannot match.

60fpsTimeline APIStaggers Scrub animationsFLIP
GSAP ScrollTrigger Plugin
GSAP Plugin

Definition: ScrollTrigger is a GSAP plugin that ties any animation to the scroll position. You can trigger, pin, scrub, or reverse animations based on when an element enters or exits the viewport.

Why I Used It

Used on every section for scroll-based reveals. The Experience section's vertical line literally "draws" as you scroll down using scrub: 0.5 — the animation plays forward/backward in sync with your scroll wheel. This level of control is impossible with pure CSS scroll animations or Framer Motion's whileInView.

Scroll-drivenScrubPin IntersectionObserver replacement
Framer Motion 11
React Animation

Definition: Framer Motion is a production-ready React animation library. It uses a declarative API — you describe what a component should look like and it handles the interpolation. Its killer feature is AnimatePresence, which animates components when they unmount (exit) from the DOM — something CSS transitions cannot do.

Why I Used It

Used for: Skills tab switching (AnimatePresence handles the old tab fading out before the new one fades in), card hover lifts (whileHover), the Open To Work badge springing in, project card hover effects, and the mobile nav drawer. It integrates naturally with React's component lifecycle unlike GSAP which needs manual cleanup.

AnimatePresencewhileHoverwhileTap whileInViewlayout animations
Lenis (Smooth Scroll)
Scroll Engine

Definition: Lenis is a lightweight smooth scrolling library by Studio Freight. It intercepts native browser scroll events and replaces them with a lerp-based (linear interpolation) smooth movement. The result is the premium "buttery" scroll feel seen on high-end agency sites.

Why I Used It

Native browser scroll is instant and jerky. CSS scroll-behavior: smooth only affects anchor links. Lenis creates a scroll experience that feels physically weighted — like momentum. It's also wired into GSAP's ticker so ScrollTrigger animations stay perfectly in sync with the smooth scroll position, not the native one.

Section 03
Custom Hooks

Reusable logic extracted into hooks — keeps components clean and DRY.

useGsap.js
Custom Hook

What it does: Creates a GSAP context scoped to a container ref. Any animations defined inside automatically target only that component's DOM, preventing selector collisions. On unmount it calls ctx.revert() to kill all animations and restore original styles — no memory leaks.

Why it matters

Without scoped contexts, a GSAP selector like ".card" would target every element with that class on the page. With gsap.context(fn, containerRef), it only targets elements inside that specific component. This is the correct pattern for GSAP in React.

// Usage pattern
const containerRef = useGsap((ctx) => { gsap.from('.card', { opacity: 0, y: 30, stagger: 0.1 }) }) // cleanup handled automatically on unmount return <section ref={containerRef}>...</section>
useSmoothScroll.js
Custom Hook

What it does: Returns a scrollTo(target) function that uses the global Lenis instance stored at window.__lenis. Falls back to native smooth scroll if Lenis isn't ready yet. Used by Navbar, Footer, and CTA buttons.

Why it matters

Instead of passing the Lenis instance down through props (prop drilling) or reaching into a Context, any component can call useSmoothScroll() and get a ready-to-use scrollTo. Single source of truth.

useTypewriter.js
Custom Hook

What it does: Cycles through an array of strings, typing characters one by one, pausing at full word, then deleting character by character before moving to the next word. Returns the current text string and an isDeleting boolean.

Why it matters

Powers the rotating title in the Hero badge: "Full Stack AI Engineer → MERN Stack Developer → React Specialist…" This shows recruiters range without cluttering the UI. The hook is pure logic — the component only renders the returned string, keeping concerns separated.

Data Architecture
Separation of Data from UI
All portfolio content — projects, experience, skills — lives in /src/data/ as plain JavaScript objects. Components import this data and render it. To update the portfolio content (e.g. add a new project), you only touch one data file, not the component.

Interview answer: "This is the data-driven component pattern — the same principle as a CMS. The UI doesn't care what the data is, only its shape. It makes the portfolio maintainable and scalable."

ParticleBackground.jsx

Definition: A full-viewport HTML5 Canvas element fixed behind all content. Uses requestAnimationFrame for a 60fps animation loop. 55 particles drift across the screen (wrapping at edges). Any two particles within 130px are connected with a line whose opacity fades with distance — creating a "neural network" aesthetic.

Why Canvas, not CSS/SVG?

Canvas is the only performant option for 55+ moving objects with dynamic line drawing. CSS animations can't draw lines between moving elements. SVG would create thousands of DOM elements, killing performance. Canvas batches all drawing in a single element, with zero DOM overhead. It also respects prefers-reduced-motion — the animation is disabled for accessibility.

Performance Decisions Made
  • Canvas for particles — not DOM elements. Zero reflow cost.
  • prefers-reduced-motion respected — canvas animation disabled for accessibility.
  • pointer-events: none on all decorative overlays — never blocks clicks.
  • GSAP context cleanup on every component unmount — no memory leaks.
  • IntersectionObserver for section detection — much cheaper than scroll event listeners.
  • Lenis smoothTouch: false — native touch scroll on mobile (better UX than lerp on touchscreen).
  • Google Fonts preconnect — DNS resolved before CSS is even parsed.
  • Passive scroll listeners — { passive: true } on all scroll handlers for jank-free scrolling.
Section 07
Interview Q&A

Ready-made answers to the most common technical questions a recruiter or interviewer will ask.

Why did you use both GSAP and Framer Motion? Isn't that redundant?
They serve completely different purposes. GSAP operates outside React's render cycle, making it ideal for performance-critical, timeline-based animations like the Hero load sequence or scroll-driven effects. Framer Motion is React-aware and handles component lifecycle animations — like animating a component out of the DOM (AnimatePresence) or hover/tap states that respond to React state. Using GSAP for everything would require manual DOM refs everywhere; using Framer Motion for everything would sacrifice performance on complex timelines.
What is Lenis and why not just use CSS scroll-behavior: smooth?
CSS scroll-behavior: smooth only applies to anchor link navigation — it doesn't affect regular wheel/trackpad scrolling. Lenis intercepts all scroll input and applies a lerp (linear interpolation) algorithm that creates a physically weighted, momentum-like scroll feel. It also syncs with GSAP's ticker so ScrollTrigger animations run against the smooth scroll position rather than the native choppy one. The result is the premium feel seen on high-end agency websites.
How did you prevent memory leaks with GSAP animations in React?
By using GSAP's context API — gsap.context(fn, containerRef) — inside every useEffect. The context scopes all animations to the component's DOM. In the useEffect cleanup I call ctx.revert(), which kills all animations created in that context and restores original CSS values. Without this, killed components would still have running tweens referencing detached DOM nodes, causing memory leaks and console errors.
Why use Canvas for the particle background instead of CSS animations or SVG?
CSS animations can't draw dynamic lines between moving elements. SVG would require creating and updating DOM nodes for every particle and line on every frame — at 55 particles with potentially hundreds of connecting lines, that's thousands of DOM mutations per second, causing severe layout thrashing. Canvas batches all drawing in a single bitmap element with zero DOM overhead. The entire animation lives in one requestAnimationFrame loop.
What is the data-driven component pattern you used?
All portfolio content — projects, experience, skills — lives in plain JavaScript files under /src/data/ as exported arrays of objects. Components import this data and render it. The component only knows the shape of the data (what fields to expect), not the content. This means updating the portfolio — adding a new project, changing a job description — requires editing only one data file, not touching any component. It's the same principle as a headless CMS.
How does the custom cursor work technically?
Two absolutely positioned divs: a dot that uses transform to snap directly to mouse coordinates (no lag), and a ring that uses requestAnimationFrame with lerp (rx += (mx - rx) * 0.12) to lag behind. The ring scales up on hoverable elements via mouseover/mouseout events that check if the target matches 'a, button, input, etc.' It's hidden on touch devices via ontouchstart and pointer: coarse media query checks. mixBlendMode: difference on the dot makes it invert colors under light backgrounds.
Why Vite over Create React App or Next.js?
Create React App uses Webpack which is slow to start and slow to HMR. Vite uses native ES Modules — the dev server starts in under 300ms and HMR is near-instant. Next.js adds SSR, file-based routing, and server components — none of which a static portfolio needs. Next.js would be the right choice if the portfolio had a blog with dynamic routes, server-side data fetching, or needed SEO at scale. For this use case, Vite gives the same developer experience with less complexity.
What are Open Graph tags and why did you add them?
Open Graph is a protocol (originally by Facebook) that controls how URLs appear when shared on social platforms — LinkedIn, Slack, WhatsApp, Twitter/X. Without OG tags, sharing your URL shows just the raw URL or a generic snippet. With og:title, og:description, and og:image, it renders a professional preview card with your name, description, and a custom image. For a portfolio, this matters because recruiters share URLs with hiring managers — the first impression of your site often happens in a Slack message, not a browser.
